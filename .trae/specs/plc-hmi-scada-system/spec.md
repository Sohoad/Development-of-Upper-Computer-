# S7-1200 PLC 上位机监控系统 Spec

## Why
工业现场需要一款与西门子 S7-1200 PLC 配套的现代化上位机软件，替代传统组态软件在交互体验、视觉动效、可扩展性方面的不足，提供高效实时通信、直观数据可视化、完整工艺控制与历史分析能力。

## What Changes
- 新增基于 Electron + 前端框架的桌面上位机应用
- 实现 S7 协议通信层，支持与 S7-1200 PLC 实时数据交互
- 提供状态监视、工艺控制、手动控制、历史图表四大功能页面
- 支持多语言（中文/英文）、多级用户权限、日志审计
- 预留 OPC UA / MQTT 扩展接口
- **BREAKING**: 无（全新项目）

## Impact
- Affected specs: 无（首个 Spec）
- Affected code: 全新代码库，涉及 `src/main/`（Electron 主进程）、`src/renderer/`（渲染进程）、`src/shared/`（共享类型与常量）

## ADDED Requirements

### Requirement: 系统架构与通信层
系统 SHALL 基于 Electron 构建桌面应用，主进程负责 S7 协议通信管理，渲染进程负责 UI 展示，两者通过 IPC 桥接通信。

#### Scenario: PLC 连接建立
- **WHEN** 用户输入正确的 PLC IP 地址、机架号、槽位号并点击连接
- **THEN** 系统通过 S7 协议（ISO-on-TCP）建立连接，状态指示灯变绿，并开始周期性轮询数据

#### Scenario: 断线自动重连
- **WHEN** PLC 通信链路中断（网络故障、PLC 断电等）
- **THEN** 系统立即检测到断线，状态指示灯变红，自动启动重连机制（指数退避），并在 UI 上显示重连倒计时。重连成功后恢复数据采集。

#### Scenario: 通信状态实时监测
- **WHEN** 系统处于运行状态
- **THEN** 主进程持续监测通信延迟、丢包率、连接状态，并通过 IPC 推送至渲染进程的状态栏展示

### Requirement: 状态监视页面
系统 SHALL 提供状态监视页面作为默认首页，包含炉体示意图、实时参数面板和报警栏。

#### Scenario: 炉体示意图展示
- **WHEN** 用户打开状态监视页面
- **THEN** 页面中央展示高温铸锭炉的二维/三维示意图（使用 Three.js 或 Canvas），根据实时数据动态渲染温度分布热力效果与液位动画

#### Scenario: 实时参数面板更新
- **WHEN** PLC 数据刷新周期到达
- **THEN** 参数面板（温度、压力、功率、电流、电压等）以数字、仪表盘、实时曲线等多种形式同步更新，数值变化时带有平滑过渡动画

#### Scenario: 报警信息分级展示
- **WHEN** PLC 上报报警信号
- **THEN** 报警栏以颜色区分等级（黄色=警告、橙色=故障、红色=紧急），显示报警时间、描述、等级，点击可查看详情与确认

### Requirement: 工艺控制页面
系统 SHALL 提供工艺控制页面，支持工艺列表浏览、详情编辑、文件操作与运行控制。

#### Scenario: 工艺列表展示
- **WHEN** 用户进入工艺控制页面
- **THEN** 展示所有已加载工艺的列表，包含工艺名称、编号、总步数、运行状态（就绪/运行中/已暂停/已完成）

#### Scenario: 工艺步骤详情编辑
- **WHEN** 用户点击某条工艺
- **THEN** 展开该工艺的所有步骤，每步显示：步号、运行时间（支持秒/分钟切换）、控制模式（温度控制/功率控制），根据控制模式动态显示对应输入框（温度控制显示设定温度℃；功率控制显示功率设定%与压力设定，压力单位支持 Pa/mBar 切换）

#### Scenario: 工艺文件导入导出
- **WHEN** 用户点击导入/导出按钮
- **THEN** 系统弹出文件选择对话框，支持 JSON 与 CSV 格式的工艺文件读写，导入后自动校验数据格式并在列表中显示

#### Scenario: 工艺运行控制
- **WHEN** 用户点击启动/暂停/停止按钮
- **THEN** 系统向 PLC 发送对应工艺控制指令，UI 上运行状态实时更新。支持跳步（跳至指定步骤）和单步执行（逐步推进）模式。

### Requirement: 手动控制页面
系统 SHALL 提供手动控制页面，允许操作员直接对 PLC 点位进行读写操作。

#### Scenario: 数字量强制控制
- **WHEN** 用户在手动控制页面对某个数字量点位执行强制 ON/OFF 操作
- **THEN** 系统通过 S7 协议写入对应地址，并在 UI 上立即反馈写入结果（成功/失败提示）

#### Scenario: 模拟量写入
- **WHEN** 用户在手动控制页面对某个模拟量点位输入数值并确认写入
- **THEN** 系统校验输入范围（基于点位配置），通过 S7 协议写入，UI 反馈写入结果

#### Scenario: 按区域分组展示
- **WHEN** 用户打开手动控制页面
- **THEN** 控制点位按区域（如 I/O 区、DB 块区、M 区）分组展示，每组可折叠/展开，支持搜索过滤

### Requirement: 历史图表页面
系统 SHALL 提供历史数据查询、可视化与导出功能。

#### Scenario: 历史数据查询与可视化
- **WHEN** 用户选择时间范围、数据点位并点击查询
- **THEN** 系统从本地数据库检索历史数据，以曲线图（ECharts）和表格形式展示。支持多曲线叠加对比。

#### Scenario: 历史数据导出
- **WHEN** 用户点击导出按钮并选择格式（Excel/CSV）
- **THEN** 系统将当前查询结果导出为对应格式文件并触发下载

### Requirement: 用户权限管理
系统 SHALL 提供多级用户权限控制。

#### Scenario: 用户登录
- **WHEN** 用户输入用户名和密码登录
- **THEN** 系统验证凭据，根据角色（操作员/工程师/管理员）加载对应权限，操作员仅可查看状态和历史，工程师可操作工艺和手动控制，管理员可管理用户和系统配置

#### Scenario: 权限拦截
- **WHEN** 低权限用户尝试执行高权限操作
- **THEN** 系统阻止操作并提示"权限不足"

### Requirement: 多语言支持
系统 SHALL 支持中文和英文界面切换。

#### Scenario: 语言切换
- **WHEN** 用户在设置中切换语言
- **THEN** 系统立即更新所有 UI 文本、图表标签、报警信息为所选语言，无需重启应用

### Requirement: 日志与审计
系统 SHALL 记录用户操作、通信事件、系统异常日志。

#### Scenario: 操作日志记录
- **WHEN** 用户执行任何写操作（工艺控制、手动写入等）
- **THEN** 系统自动记录操作时间、用户、操作类型、目标点位、写入值，存入本地数据库

#### Scenario: 日志查看
- **WHEN** 管理员打开日志查看页面
- **THEN** 可按时间范围、用户、操作类型筛选日志，支持导出为 CSV

### Requirement: 响应式布局
系统 SHALL 适配不同分辨率的工业显示器（1024×768 至 3840×2160）。

#### Scenario: 窗口缩放适配
- **WHEN** 用户调整窗口大小或切换至不同分辨率显示器
- **THEN** 界面布局自适应调整，关键参数面板和图表保持可读性，不出现元素重叠或溢出

### Requirement: 扩展接口预留
系统 SHALL 在架构层面预留 OPC UA 和 MQTT 协议接入能力。

#### Scenario: 扩展协议接口
- **WHEN** 后期需要接入 MES/SCADA 系统
- **THEN** 开发人员可在通信层扩展模块中实现 OPC UA Client 或 MQTT Client，通过统一的数据总线（EventEmitter / Observable）与现有系统集成
